Detailed Description

This file defines a set of classes useful for creating histograms.

Note
July 2018: These classes are several years old and are in dire need of simplification. They do, however, catch a large number of errors, which is one reason they are so complex.

TODO: BinSpecs are stored by reference inside HistoVec. This is BAD! If the BinSpecs go out of scope before the histogram is written, undefined behavior.

Some useful features are 1 and 2-dimensional histograms which can have bins that are uniform in linear or log-space, have overflow bins, supports bin weighting and histogram division (for creating a non-trivial probability density) while respecting a bins actual boundaries within machine epsilon.

In this kdpHistogram scheme:

  • A "bin" is defined as the semi-inclusive interval [lower, upper).
  • The correct bin for value x is found by incrementing bins until (x >= lower_i) == false. Thus x belongs to the previous bin (i - 1). Since only floating point numbers are binned, (for which comparison is transitive) this definition is acceptable.

However, since the values being binned have error (either from measurement or floating point rounding error and cancellation), there does arise the question of a value which falls within Order(error) of an edge – which bin does it belong to? Perhaps its weight should be split between both. Since this is generally not a large problem, it is currently not addressed.
